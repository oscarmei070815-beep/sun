# 一、整体画面风格

```text
画面风格要求：

1. 背景为深黑色宇宙空间，不是纯黑空白。
2. 使用大量星点构成多层星场，星点大小、亮度、距离感不同。
3. 可以加入轻微蓝绿色或紫色星云雾感，但不要过度抢眼。
4. 场景中有细长流星或彗星拖尾，数量少，作为空间动态点缀。
5. 太阳系主体位于一个广阔 3D 空间中，支持近距离观察和远距离总览。
6. 行星、卫星、小行星带、轨道线、标签都应能同时存在。
7. 整体视觉应干净、深邃、科学、电影化。
8. 不要做卡通风格。
9. 不要做过度霓虹赛博朋克风。
10. 不要做 2D 平面太阳系图。
```

# 二、太阳效果

```text
太阳是整个系统的视觉中心和主要光源，必须重点实现。

太阳要求：

1. 位于太阳系中心。
2. 太阳体积明显大于所有行星。
3. 表面不是普通黄色球体，而是橙、黄、红混合的程序化等离子体纹理。
4. 太阳表面要有类似流动斑纹、噪声、火焰纹、热流纹的效果。
5. 使用 ShaderMaterial 或 CanvasTexture / procedural noise 生成动态太阳表面。
6. 太阳外层有强烈但柔和的黄色 glow。
7. glow 外侧再有橙红色 corona 日冕光晕。
8. 可添加环状或弧状 looping prominences，也就是围绕太阳附近的淡红色弧线喷发结构。
9. 使用 Bloom 或 UnrealBloomPass 增强太阳发光。
10. 太阳自身材质应使用 MeshBasicMaterial 或 ShaderMaterial，不要被普通光照变暗。
11. 太阳必须向全场提供 PointLight，成为行星和卫星明暗的真实来源。
12. 行星受光面必须朝向太阳，背光面必须变暗。
```

# 四、行星视觉细节

```text
行星不能只是纯色球体，每个都要有明显差异。

Mercury:
- 小型灰色岩石行星。
- 表面可有噪声、坑洼感或斑驳纹理。

Venus:
- 偏黄白色或淡金色。
- 有厚重大气感。
- 可加入柔和云层纹理。

Earth:
- 必须是多层结构。
- 有海洋、陆地、冰盖。
- 有实时或动态云层。
- 有蓝色大气边缘 glow。
- 夜侧出现城市灯光，昼侧不显示城市灯光。
- 有清晰昼夜分界线。
- 月球围绕地球运动。
- 月球也需要根据太阳方向产生明暗面。

Mars:
- 红橙色岩石行星。
- 可有暗色地表斑块。
- 有 Phobos 和 Deimos 两颗小卫星。

Jupiter:
- 巨大气态行星。
- 有条纹云带。
- 可加入大红斑。
- 有 Io、Europa、Ganymede、Callisto 四颗主要卫星。
- 木星尺寸要明显大于内行星。

Saturn:
- 土黄色气态行星。
- 有巨大倾斜环带。
- 环带必须是多层细密纹理，不是单一透明圆片。
- 环带应有明暗层次、条纹、透明度变化。
- 土星本体和环带都要受太阳光照影响。
- 可实现土星本体投射到环带上的阴影。
- 有 Titan、Enceladus、Rhea、Iapetus 等卫星。

Uranus:
- 淡青色或蓝绿色冰巨星。
- 几乎横躺，轴倾角接近 90 度。
- 有淡淡的环系统。
- 环带可以偏竖直或明显倾斜，体现天王星特殊倾角。
- 有 Titania 和 Oberon 卫星。
- 表面可以有很淡的青色条带。

Neptune:
- 深蓝色冰巨星。
- 可有淡色条纹。
- 有 Triton 卫星。

Pluto:
- 小型远距离矮行星。
- 有 Charon 卫星。
- 可显示为较小但可聚焦对象。
```

# 五、小行星带与柯伊伯带

```text
必须实现两个带状结构：

1. Asteroid Belt
- 位于 Mars 和 Jupiter 之间。
- 由大量白色、灰色、灰褐色小岩石组成。
- 小行星不能只是点，近距离看应有不规则岩石块感。
- 使用 InstancedMesh 实例化渲染。
- 至少生成 1000 个小行星实例。
- 每个小行星应有随机：
  - 半径
  - 角度
  - 轨道高度偏移
  - 形状缩放
  - 旋转角
  - 自转速度
  - 材质亮度
- 小行星带整体呈横向环带，在远景中形成明显的白色碎石带。
- 小行星带应缓慢旋转。

2. Kuiper Veil
- 位于 Neptune / Pluto 外侧。
- 比小行星带更远、更暗、更稀疏。
- 可使用 Points 或 InstancedMesh。
- 呈淡淡的外层粒子环。
- 有标签 “Kuiper Veil”。
```

# 七、运动与比例

```text
运动系统要求：

1. 所有行星围绕太阳公转。
2. 所有行星自身自转。
3. 所有卫星围绕对应母星公转。
4. 母星移动时，卫星系统要随母星整体移动。
5. 小行星带整体缓慢旋转。
6. 彗星或流星可以沿长椭圆轨道或随机深空路径运动。
7. 运动必须平滑，不要像静态球体随机漂浮。
8. 使用 requestAnimationFrame。
9. 使用 THREE.Clock 的 delta time 更新动画。
10. 不要让动画依赖固定帧率。

比例要求：

1. 不要求天文绝对真实比例，因为真实比例会导致画面不可读。
2. 但必须让比例“感觉准确”：
   - 太阳最大。
   - 木星、土星明显大于内行星。
   - 内行星靠近太阳。
   - 外行星距离更远。
   - 小行星带在 Mars 和 Jupiter 之间。
   - Kuiper Veil 在 Neptune / Pluto 外侧。
3. 行星尺寸可以适度夸张。
4. 行星距离可以压缩。
5. 但空间层级必须清晰。
6. 相机拉远时，应能看出完整太阳系结构。
7. 相机靠近时，应能看出行星、卫星、环带、光照细节。
```

# 八、自然光照与明暗

```text
光照必须来自太阳方向，不能用固定方向光随便照亮。

要求：

1. 太阳是主光源。
2. 使用 PointLight 从太阳位置发光。
3. 行星朝向太阳的一侧明亮。
4. 行星背向太阳的一侧变暗。
5. 地球夜侧显示城市灯光。
6. 地球昼侧不显示城市灯光。
7. 月球、火卫、木卫、土卫等也应有太阳方向决定的明暗。
8. 土星环要有受光面和背光面。
9. 可以添加很弱的 AmbientLight，但不能破坏明显昼夜分界。
10. 行星明暗边界应自然，不要硬切。
11. 可以使用自定义 shader 实现 terminator 昼夜分界。
```

# 十一、交互与相机

```text
相机系统要求：

1. 支持 OrbitControls。
2. 用户可以拖拽旋转视角。
3. 用户可以滚轮缩放。
4. 用户可以平移视角。
5. 点击或从下拉框选择任意天体后，相机平滑飞行到该天体附近。
6. 聚焦时，相机继续跟随该天体运动。
7. 选择 Free flight / manual camera 后，取消跟随，用户自由控制。
8. Cinematic Tour 按钮启动自动电影镜头：
   - 先飞近太阳。
   - 再掠过内行星。
   - 再穿越小行星带。
   - 再看木星和木卫系统。
   - 再看土星环和土卫。
   - 再看天王星倾斜环。
   - 最后拉远到完整太阳系。
9. Deep-Space View 按钮把相机拉远，显示太阳、行星、小行星带、柯伊伯带和星场整体结构。
10. 相机飞行动画要平滑，可以使用 lerp、slerp 或 tween。
11. 聚焦天体时，不能瞬间跳转。
12. 聚焦不同尺度对象时，相机距离要自动适配：
   - 太阳距离远一些。
   - 行星距离适中。
   - 卫星距离近一些。
   - 小行星带和柯伊伯带使用广角远景。
```

   - 参考图：4a371fb3-*（天王星聚焦时仍能看到顶部木土星一排）、946a35b8-*（天王星聚焦时背景有 Mercury/Venus/Earth/Jupiter/Saturn 一字排开）。

2. Deep-Space View 的具体构图：
   - 相机位于黄道面附近（略微抬高 15°–25°），行星近似一字排开。
   - 从画面右侧太阳位置向左延伸，依次经过 Mercury、Venus、Earth、Mars、Asteroid Belt、Jupiter、Saturn、Uranus、Neptune、Pluto、Kuiper Veil。
   - 所有主要天体的标签必须同时在画面内可读，小行星带呈横向白色碎石带穿过画面。
   - 不要采用高角度俯视（会破坏"一字排开"的可读性）。
   - 参考图：b18b2f6c-*。

3. 土星环的细节强度：
   - 环面必须肉眼可辨至少 3–4 条主环（对应 A 环、B 环、C 环）以及一条暗缝（Cassini gap）。
   - 实现方式：RingGeometry + 自定义 shader 或 CanvasTexture 生成径向条纹（alpha / brightness 随半径呈多带状变化，而不是单一 radial gradient）。
   - 土星本体必须投射阴影到环面（或通过 shader 近似：环上靠近本体背光侧的扇形区域变暗）。
   - 土星本体基础色必须是土黄/米黄（#d8b87a 附近），不要因为 bloom 过强看起来纯白（控制 bloom threshold）。
   - 参考图：797fa690-*、893d85df-*、c89cbc9a-*。

4. 天王星的姿态与环：
   - 轴倾角 97.77°，即接近"躺着自转"。环面法向近似水平。
   - 从黄道面方向观察时，天王星的环看起来是近似"正圆正对镜头"（从上方俯视才看得到扁椭圆）。
   - 环为淡蓝灰色，非常纤细，多层但不抢戏，透明度明显低于土星环。
   - Titania 和 Oberon 在环的两侧，近距离观察时清晰可见并带标签。
   - 参考图：5c26653f-*、6dc6d358-*、c89cbc9a-*。

5. 太阳的多层视觉结构：
   - 从内到外至少 4 层：核心实体球（程序化等离子体纹理，含暗色对流胞/黑子斑块）→ 第一层 corona（黄白）→ 第二层 corona（橙）→ 第三层 corona（深橙红）。
   - 每一层 corona 使用 additive blending 的 Sprite 或球壳 mesh，半径递增，alpha 递减。
   - 表面纹理必须有流动感（uniform time 驱动 fbm noise），而不是静态贴图。
   - 可选：2–3 条围绕赤道附近的 looping prominence 弧（淡红色弯曲管状结构，非必需但加分）。
   - UnrealBloomPass threshold 约 0.3–0.5、strength 1.2–1.8，让太阳发光但不让土星/地球烧白。
   - 参考图：e40f288f-*。

6. UI 颜色规范（与截图一致）：
   - 控制面板背景：`rgba(12, 18, 34, 0.82)`，细边框 `rgba(125, 160, 255, 0.18)`。

   - 复选框（Orbits/Labels/Bloom/Pause）：打勾时为橙黄色勾选框 + 淡灰标签文字。

7. 标签显示策略：

   - 标签位置在天体上方或右上方（根据相机视角自动），不要遮挡天体本体。
